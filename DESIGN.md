# DESIGN.md

## Visual Direction

The website should use a black-and-white design.

The style should feel:

* clean
* modern
* analytical
* professional
* minimal
* classy and data focused

## Color Palette

Use only black, white, and grayscale tones.

Preferred colors:

* Background: `#050505`
* Main text: `#f5f5f5`
* Secondary text: `#b5b5b5`
* Muted text: `#777777`
* Card background: `#111111`
* Border color: `#2a2a2a`
* Button background: `#f5f5f5`
* Button text: `#050505`

Do not use colorful accents unless explicitly requested.

## Layout

Use a one-page scrolling layout with clear sections:

* Header
* Hero
* About
* Skills
* Projects
* Contact
* Footer

The layout should be mobile-first and responsive.

## Design Style

* Lots of whitespace
* Clear typography
* Sharp section separation
* Simple cards
* Thin borders
* Subtle hover effects
* No colorful gradients
* No playful animations
* No overly decorative elements

## Hero Section

The hero section should feel strong and focused.

Preferred hero text style:

* large name/title
* short tagline
* one short supporting sentence
* clear call-to-action buttons

The hero should not feel crowded.

## Cards

Project and skill cards should use:

* dark background
* thin gray border
* subtle hover border change
* clean spacing
* readable text

## Typography

Use a modern sans-serif font.

Good options:

* Inter
* system font stack
* Arial / Helvetica fallback

Text should be easy to read on dark background.

## Accessibility

* Keep contrast high
* Use semantic headings
* Buttons and links must be keyboard accessible
* Do not rely only on hover states
* Images should have alt text

## Overall Feeling

The final website should feel like a serious technical portfolio for someone working with data, systems, infrastructure, and reliability.
